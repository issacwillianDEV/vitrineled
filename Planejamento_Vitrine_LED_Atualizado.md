# Planejamento Atualizado — Landing Page "Vitrine LED Veneza" (Pré-cadastro)

Documento consolidado com base no planejamento inicial e nos ajustes solicitados em conversa de 09/02/2026 a 11/02/2026.

## 1) Objetivo da página
Criar uma landing page única (LP), focada no projeto **Vitrine LED Veneza**, para:

- Apresentar a proposta com clareza e impacto
- Mostrar planos, diferenciais e condições comerciais
- Capturar pré-cadastro de empresas (**sem cobrança nesta etapa**)
- Acompanhar metas de validação antes da implantação

Metas:

- **30 empresas pré-cadastradas** para iniciar implantação
- **10 empresas Premium/VIP** (Núcleo Prime LED)

Mensagem principal:

- **"Se a empresa não está aqui, não está sendo vista."**

Posicionamento complementar:

- **"Na Vitrine do Bairro"** (linguagem para cativar comerciantes da região)

## 2) Escopo de entrega

### 2.1 Landing Page (1 página)
- Página única, responsiva (mobile-first)
- CTA visível: pré-cadastro / ver planos / WhatsApp

### 2.2 Formulário de pré-cadastro (sem cobrança)

### 2.3 Contadores de meta (30 e 10)
- Total pré-cadastradas (X/30)

- Exibição opcional na LP ou apenas no painel admin

### 2.4 Painel administrativo simples
- Acompanhamento em tempo real das inscrições
- Filtros e exportação para operação comercial

- Não limitar a uma única imagem de BR específica

- Priorizar materiais em **alta definição** para hero e criativos em vídeo

## 9) Experiência Premium no Scroll (nível alto padrão)
- Construir uma experiência visual avançada, com sensação de site premium
- Aplicar **scrollytelling**: conforme o usuário rola a página, imagem/vídeo evoluem com narrativa visual
- Criar transições cinematográficas entre blocos (hero -> proposta -> planos -> CTA)
- Usar animações com propósito comercial (revelar benefícios, provas e urgência), sem poluição visual
- Incluir trecho de vídeo \"top\" na abertura e/ou seções estratégicas, com foco em impacto e credibilidade
- Garantir versão mobile de alto nível, mantendo fluidez e legibilidade
- Trabalhar otimização de performance (compressão, lazy-load, fallback de vídeo/imagem) para não perder conversão
- Padrão de implementação sugerido: animações avançadas via timeline (ex.: GSAP/ScrollTrigger), com fallback para navegadores/dispositivos mais limitados
- Resultado esperado: percepção de marca premium, maior tempo na página e aumento da taxa de pré-cadastro

## 10) Stack recomendada (fase 1 - foco front-end)
- Base: **Next.js (App Router) + React + TypeScript**
- Scroll avançado: **GSAP + ScrollTrigger**
- Estilização e consistência visual: **Tailwind CSS**
- Suavização de scroll (opcional): **Lenis**
- Microinterações de UI (opcional): **Framer Motion**
- Otimização: code splitting, lazy load de vídeos/imagens e compressão de assets
- Observação de escopo: nesta fase, **sem banco de dados** (apenas front-end premium)
